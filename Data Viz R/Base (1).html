Our mission: catch them all!

A very common issue to every parent is how to advice their 10-year-old child when the kid is about to leave on a world-wide trip. There are many important questions to be adressed before the trip, like “How many pokemons there are?”, “Who is the strongest?” “How do I catch them all?”.

On this document we try to provide some insights on the pokemon world to help parents to advice their children on how to catch them all. We will be using the pokemon dataset from the tidytuesday repository, which contains information about all the pokemons from the first generation to the seventh generation.

Not all of these features contain valuable information for our analysis, so we will be selecting only the ones that we think are relevant:

  • id: the id of the pokemon
  • pokemon: the name of the pokemon
  • height: the height of the pokemon in meters
  • weight: the weight of the pokemon in kilograms
  • base_experience: the base experience of the pokemon when catched
  • type_1: the primary type of the pokemon
  • type_2: the secondary type of the pokemon (if it has one)
  • hp: the health points of the pokemon
  • attack: the attack points of the pokemon
  • defense: the defense points of the pokemon
  • special_attack: the special attack points of the pokemon
  • special_defense: the special defense points of the pokemon
  • speed: the speed points of the pokemon
  • generation_id: the generation on wich the pokemon was introduced
Code 
# drop columns that we don't need
pokemon_df <- pokemon_df |>
  select(-c(species_id, url_icon, url_image))

One of the first things we notice is that there are some missing values (NA) on the generation_id variable (exactly 147). After some research on this subset we were able to determine that these pokemons are all spetial edition pokemons like Charizard Mega X or Pikachu Pop Star. In consequence we will be dropping these pokemons from our analysis, since they are not representative of the general pokemon population and they could mislead our insights.

Code 
# drop pokemons with missing generation_id
pokemon_df = pokemon_df |> 
  filter(!is.na(generation_id))

Now we have a clean dataset with 802 pokemons and 19 variables.

As we will be performing some analysis on the types of pokemons it’s important to get familiar with these variables. During this process we noticed that there are some pokemons that have a secondary type and some that don’t (400 over 802). For the ones that don’t have a secondary type, the value of type_2 is NA. We will be replacing these NA values with “none” to make our analysis easier.

Code 
#manage NAs in type 2
pokemon_df = pokemon_df |> 
  mutate(pokemon = str_to_title(pokemon), #gives some format to Pokemon names.
         type_2 = ifelse(is.na(type_2), "none", type_2) |> str_to_title(),
         type_1 = str_to_title(type_1))

Pokemon distribution by generation

Now is time to adress some of our questions: How many pokemons there are in each generation? How powerfull are they? We can get some ideas about these topics by looking at the distribution of pokemons.

Code 
# to a better analysis we create to combined variables
pokemon_df = pokemon_df |> 
  mutate(comb_attack = attack + special_attack,
         comb_defense = defense + special_defense)
         
#summarise pokemon distribution by generation and main stats
interest_stats = pokemon_df |> 
  group_by(generation_id) |> 
  summarise(n = n(),
            avg_hp = mean(hp, na.rm = TRUE),
            avg_attack = mean(comb_attack, na.rm = TRUE),
            avg_defense = mean(comb_defense, na.rm = TRUE),
            avg_speed = mean(speed, na.rm = TRUE)
)

From this we can see that the most populated generations are the 1st, 3rd and 5th. Nevertheless, the most powerfull generation considering all four stats (combined attack, combined defense, hp and speed) is the 4th generation.

What can we say about types

Pokemons come on a wide variety of types. Does it matter to pick a pokemon of a certain type? Are some types more common than others? Are some types more powerful than others? These are very important questions as we know that advantages and weaknesses of types are a key element of the pokemon world. For example, water type pokemons are strong against fire type pokemons but weak against electric type pokemons. In consequence, if we want to catch them all we need to know which types are more common and which types are more powerful.

Code 
#table for count of pokemon by type and generation 
pokemon_df |> 
  group_by(generation_id, type_1) |> 
  summarise(n = n()) |> 
  pivot_wider(names_from = generation_id, values_from = n, values_fill = 0) |> 
  mutate(Total = rowSums(across(where(is.numeric)))) |>
  kable() %>%
  kable_styling(bootstrap_options = c("striped", "hover", "condensed", "responsive"),
                full_width = F, position = "center")

Type proportion on our sample

From table Table 2 we can see that the most common types are water, normal and Grass. However, these types are not equally distributed across generations. For example, the water type is more common in the 1st generation than in the 4th generation and Normal type is the rarest main type on the 6th generation.

The analysis can be enriched if we consider the secondary type of pokemons. Considerin the 18 main types and the 19 secondary types (including “None”) there are 342 possible combinations we can create a heatmap to visualize the proportion of pokemons by type 1 and type 2 across the total universe of registered pokemons. This will allow us to see which combinations of types are more common and which are more rare.

Code 
# count of pokemon by type 1 and type 2
poke_type <- pokemon_df|> 
  group_by(type_1, type_2) |> 
  summarise(n = n(), .groups = "drop") |>
  complete(type_1, type_2, fill = list(n = 0)) |>
  ungroup() |> 
  mutate(prop = n * 100 / sum(n), 
         norm_prop = case_when (prop==0 ~ -2.1,
                                TRUE ~ log(prop))
        )

poke_type <- poke_type |>
  mutate(type_2 = fct_relevel(type_2, "None")) #we make it a factor so "None" appears first

#graph 
ggplot(poke_type, aes(x = type_2, y = type_1, fill = norm_prop)) +
  geom_tile(color = "lightgrey") +
  scale_fill_gradient(low = "white", high = "#B22222") +
  labs(
    title = "Normalized proportion of pokemons by types",
    x = "Secondary Type",
    y = "Main Type",
    fill = "Norm. Prop.",
    caption = "Normalization is performed as ln(proportion) to better visualize types with low proportions."
  ) +
  theme_minimal() +
  theme(axis.text.x = element_text(angle = 45, hjust = 1))

Assuming that each pokemon species distributed homogeneously across the world, meaning that for example catching a Blastoise is as likely as catching a Squirtle (strong assumption), we could say that the most abuntant pokemons belong to the Water-None combination. Outside the None secondary type, the most common combinations are Normal-Flying (like Pidgey), Grass-Poison (like Bulbasaur) and Bug-Flying (like Butterfree). On the other hand, there are 42 combinations that have only one pokemon species, like the Ghost-Fairy, Fire-Dark, or Dragon-Electric.
